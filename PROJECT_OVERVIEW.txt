PROJECT OVERVIEW
----------------
AEVA is a comprehensive Python-based platform for algorithm evaluation and 
validation, implementing an "Evaluation-as-a-Service" (EaaS) approach with 
four core subsystems:

1. AEVA-Guard   - Quality gates and delivery protection
2. AEVA-Bench   - Standardized evaluation benchmarks  
3. AEVA-Auto    - Automation pipeline orchestration
4. AEVA-Brain   - Intelligent AI-powered analysis (using Claude LLM)

CORE FEATURES
-------------
✓ Automated quality gate enforcement
✓ Multi-dimensional metrics evaluation
✓ Parallel benchmark execution
✓ Intelligent root cause analysis using LLM
✓ Pipeline orchestration with retry logic
✓ REST API with FastAPI
✓ Docker containerization support
✓ Comprehensive test coverage

QUICK START
-----------
1. Install:
   pip install -r requirements.txt
   pip install -e .

2. Configure:
   cp .env.example .env
   # Add AEVA_BRAIN_API_KEY=your_claude_api_key

3. Run example:
   python examples/basic_usage.py

4. Start API server:
   aeva server
   # Visit http://localhost:8000/docs

5. Run tests:
   pytest tests/ -v

DOCKER DEPLOYMENT
-----------------
docker-compose up -d              # Start all services
docker-compose logs -f aeva-api   # View logs
docker-compose down               # Stop services

USAGE EXAMPLE
-------------
from aeva import AEVA
from aeva.guard import ThresholdGate

# Initialize platform
aeva = AEVA(config_path="config/aeva.yaml")

# Create evaluation pipeline
pipeline = aeva.create_pipeline(name="model_evaluation")

# Add quality gate
gate = ThresholdGate("accuracy_gate", threshold=0.90, metric_name="accuracy")
aeva.guard.add_gate(gate)

# Run evaluation
result = aeva.run(pipeline, algorithm=your_model)

# Check results
print(result.summary())
if result.gate_result.passed:
    print("✓ Quality gate passed - ready for deployment")
else:
    print("✗ Quality gate failed - needs improvement")
    if result.analysis:
        print(f"Root causes: {result.analysis.root_causes}")
        print(f"Recommendations: {result.analysis.recommendations}")

CONFIGURATION
-------------
Main config: config/aeva.yaml
Environment: .env file

Key settings:
- brain.api_key: Claude API key (required for Brain module)
- guard.default_threshold: Quality gate threshold (0.85)
- bench.parallel_execution: Enable parallel benchmarks (true)
- auto.worker_concurrency: Number of workers (4)
